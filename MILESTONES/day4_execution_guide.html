df_benchmark = pd.DataFrame(africa_benchmark)

# Payment Readiness Score — composite index
# Weighted: mobile money 40%, internet 25%, smartphone 20%, banked 15%
df_benchmark["payment_readiness_raw"] = (
    df_benchmark["mobile_money_penetration_pct"] * 0.40 +
    df_benchmark["internet_penetration_pct"]     * 0.25 +
    df_benchmark["smartphone_penetration_pct"]   * 0.20 +
    df_benchmark["banked_population_pct"]        * 0.15
)

# Normalize to 0-100
min_val = df_benchmark["payment_readiness_raw"].min()
max_val = df_benchmark["payment_readiness_raw"].max()
df_benchmark["payment_readiness_score"] = (
    (df_benchmark["payment_readiness_raw"] - min_val) /
    (max_val - min_val) * 100
).round(1)

print(df_benchmark[["country",
                     "mobile_money_penetration_pct",
                     "internet_penetration_pct",

    Cell 2 — Align time frequencies
    Crypto data is daily. Demand data is weekly. You cannot merge on date directly — you must resample crypto to weekly first. This is a core data engineering concept: aligning temporal granularity before joining.
    python — cell 2
    # Resample daily crypto data to weekly # Price: last price of the week (closing) # Volume: sum over the week (total traded) # Momentum: last value (most recent signal) df_crypto_weekly = ( df_crypto .set_index("date") .resample("W") .agg({ "bitcoin_price_usd": "last", "bitcoin_volume_usd": "sum", "ethereum_price_usd": "last", "ethereum_volume_usd": "sum", "CMI_index": "last", "MSI_index": "last", }) .reset_index() .rename(columns={"date": "week"}) ) print(f"Crypto resampled to weekly: {df_crypto_weekly.shape}") print(f"Date range: {df_crypto_weekly.week.min().date()} to {df_crypto_weekly.week.max().date()}") print(df_crypto_weekly.head(3).to_string(index=False))
    Why agg differently per column?
    Price → "last": You want the closing price at the end of the week, not the average. Exchanges report closing prices.
    Volume → "sum": Volume accumulates. Total weekly volume is the sum of all daily volumes.
    CMI/MSI → "last": These are index scores. The most recent value is what matters for the current week's market condition.

    Choosing the wrong aggregation function corrupts your analysis silently — no error, wrong numbers. This is one of the most common mistakes in data engineering. Always ask: what does this column represent and what operation preserves its meaning across time periods?
