Concepts you need before you start
API
Application Programming Interface. Your code sends a request to a server over the internet. The server sends back data. No scraping, no downloading files manually.
JSON
The format APIs return data in. Looks like a Python dictionary. You parse it with .json() and extract the keys you need.
Status code 200
Means the request succeeded. 429 = rate limited (too many calls). 404 = endpoint not found. Always check this before using the response.
Rate limit
CoinGecko free tier allows ~10-50 calls/minute. Exceed it and you get a 429 error and a temporary block. Use time.sleep() to stay safe.
Rolling average
The average of the last N values at each point. A 7-day rolling average smooths out daily noise to show the trend direction.
NaN
Not a Number. Rolling calculations can't compute an average until they have enough rows. The first N rows become NaN. You must handle these before saving.

    Cell 3 — Write the data-pull function
    Insert a new code cell. Type this entire function — do not skip the docstring. Read each block as you type it and understand what it does:
    python — cell 3
    def get_coin_data(coin_id, days=365):
    """
    Pull daily price and volume data from CoinGecko.
    coin_id: "bitcoin" or "ethereum"
    days: number of days of history to pull
    Returns a pandas DataFrame
    """
    url = f"https://api.coingecko.com/api/v3/coins/{coin_id}/market_chart"

    params = {
        "vs_currency": "usd",
        "days": days,
        "interval": "daily"
    }

    # Make the API request
    response = requests.get(url, params=params)

    # Check if the request was successful
    if response.status_code != 200:
        print(f"Error: {response.status_code} - {response.text}")
        return None

    # Parse the JSON response
    data = response.json()

    # Extract prices and volumes
    prices = data["prices"]            # list of [timestamp, price]
    volumes = data["total_volumes"]    # list of [timestamp, volume]

    # Convert to DataFrame
    df_price = pd.DataFrame(prices, columns=["timestamp", f"{coin_id}_price_usd"])
    df_volume = pd.DataFrame(volumes, columns=["timestamp", f"{coin_id}_volume_usd"])

    # Merge price and volume on timestamp
    df = pd.merge(df_price, df_volume, on="timestamp")

    # Convert timestamp from milliseconds to readable date
    df["date"] = pd.to_datetime(df["timestamp"], unit="ms").dt.date
    df = df.drop("timestamp", axis=1)

    # Put date first
    df = df[["date", f"{coin_id}_price_usd", f"{coin_id}_volume_usd"]]

    return df

print("Function defined. Ready to pull data.")
    Mentor note — why a function, not raw code?
    You are calling this logic twice — once for Bitcoin, once for Ethereum. A function means you write the logic once and reuse it. If there is a bug or the API changes, you fix it in one place. This is the difference between writing code and writing good code. Every analyst who gets hired writes in functions. Every student writes the same block of code three times.

    Cell 6 — Engineer 4 new market intelligence columns
    Raw price and volume data alone is not enough for analysis. You need derived signals that capture trend direction and momentum. These 4 columns are what you will use in the scoring model in Week 3:
    python — cell 6
    # Convert date to datetime for rolling calculations
df_crypto["date"] = pd.to_datetime(df_crypto["date"])

# 7-day rolling average of BTC price (short-term trend)
df_crypto["btc_7day_avg"] = df_crypto["bitcoin_price_usd"].rolling(7).mean()

# 30-day rolling average of BTC price (medium-term trend)
df_crypto["btc_30day_avg"] = df_crypto["bitcoin_price_usd"].rolling(30).mean()

# Price momentum: % change over last 30 days
df_crypto["btc_30d_change_pct"] = df_crypto["bitcoin_price_usd"].pct_change(30) * 100

# Volume momentum: 7-day rolling average volume
df_crypto["btc_volume_7day_avg"] = df_crypto["bitcoin_volume_usd"].rolling(7).mean()

print("New columns added:")
print(df_crypto.columns.tolist())
print(f"\nNulls introduced by rolling calculations:")
print(df_crypto.isnull().sum())

    Mentor note — why these 4 columns?
    btc_7day_avg — smooths daily noise. Tells you the short-term price direction.
    btc_30day_avg — tells you the medium-term market regime. Is crypto in a bull or bear phase?
    btc_30d_change_pct — momentum signal. When BTC rises 20%+ in 30 days, P2P volumes in Africa spike. This is a leading indicator for exchange activity.
    btc_volume_7day_avg — smoothed volume. Raw daily volume is noisy. A 7-day average shows whether trading activity is genuinely increasing.

    These are the same signals professional analysts at crypto exchanges track daily. You are not doing a school project — you are building a real market intelligence dataset.
