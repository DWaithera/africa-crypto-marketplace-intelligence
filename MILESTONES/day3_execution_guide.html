Concepts you need before you start
pytrends
An unofficial Python wrapper for Google Trends. It scrapes the same data you see on trends.google.com — no API key needed, but it has strict rate limits.
Google Trends scale (0–100)
Google does NOT return raw search counts. It returns a relative interest score. 100 = peak interest in that period. 50 = half the peak. 0 = not enough data. Always relative, never absolute.
geo code
Google Trends uses ISO country codes: KE = Kenya, NG = Nigeria, GH = Ghana, ZA = South Africa. Pass these to pytrends to get country-specific data.
timeframe
"today 52-w" means the last 52 weeks from today. This gives weekly data points — one per week. Do not use "today 12-m" — it gives monthly data which is too coarse.
429 error
Google has blocked your IP temporarily for too many requests. It is not an error in your code. Wait 60 seconds minimum, then retry only the failed call. Never retry in a loop.
Demand Index
A composite score you build by averaging the search interest across all 3 keywords (bitcoin, USDT, crypto) per country per week. One number that represents total crypto demand.

    Cell 2 — Write the pull function with rate limit protection
    This function is more defensive than the CoinGecko one. Google Trends will block you faster. Read the retry logic carefully — it is not optional.
    python — cell 2
    def pull_trends(keyword, geo_code, country_name, retries=3):
    """
    Pull Google Trends data for one keyword in one country.
    Returns a DataFrame with date and interest score (0-100).
    Retries up to 3 times on failure with increasing wait.
    """
    for attempt in range(retries):
        try:
            # Build the payload — tell pytrends what to search
            pytrends.build_payload(
                kw_list=[keyword],
                geo=geo_code,
                timeframe=TIMEFRAME
            )

            # Pull interest over time
            df = pytrends.interest_over_time()

            if df.empty:
                print(f"  No data returned for {keyword} in {country_name}")
                return None

            # Clean up
            df = df.reset_index()
            df = df.rename(columns={keyword: "interest", "date": "week"})
            df = df[["week", "interest"]]
            df["keyword"]  = keyword
            df["country"]  = country_name
            df["geo_code"] = geo_code

            return df

        except Exception as e:
            wait = 60 * (attempt + 1)   # 60s, 120s, 180s
            print(f"  Attempt {attempt+1} failed: {e}")
            print(f"  Waiting {wait} seconds before retry...")
            time.sleep(wait)

    print(f"  FAILED after {retries} retries: {keyword} / {country_name}")
    return None

print("Function defined.")

    Cell 6 — Build the Demand Index per country per week
    The raw data has 3 separate rows per country per week (one per keyword). You need to collapse them into one row with one demand score. You do this by averaging the interest across all 3 keywords. That average IS your Demand Index.
    python — cell 6
    # Pivot: one row per country per week, one column per keyword
df_pivot = df_raw.pivot_table(
    index=["week", "country"],
    columns="keyword",
    values="interest"
).reset_index()

# Flatten column names
df_pivot.columns.name = None
df_pivot = df_pivot.rename(columns={
    "bitcoin": "interest_bitcoin",
    "USDT":    "interest_USDT",
    "crypto":  "interest_crypto"
})

# Build Demand Index: equal weight average of all 3 keywords
df_pivot["demand_index_raw"] = df_pivot[[
    "interest_bitcoin",
    "interest_USDT",
    "interest_crypto"
]].mean(axis=1)

# Normalize to 0–100 per country (so countries are comparable)
def normalize_0_100(series):
    min_val = series.min()
    max_val = series.max()
    if max_val == min_val:
        return series * 0
    return (series - min_val) / (max_val - min_val) * 100

df_pivot["demand_index"] = df_pivot.groupby("country")["demand_index_raw"].transform(
    normalize_0_100
).round(2)

print(f"Demand Index shape: {df_pivot.shape}")
print(f"Columns: {df_pivot.columns.tolist()}")
print(f"\nSample — Kenya:")
print(df_pivot[df_pivot.country == "Kenya"].tail(5))
    Expected output
    Demand Index shape: (208, 7)
    Columns: ['week', 'country', 'interest_bitcoin', 'interest_USDT', 'interest_crypto', 'demand_index_raw', 'demand_index']

    Sample — Kenya:
      week       country  interest_bitcoin  interest_USDT  interest_crypto  demand_index_raw  demand_index
      2025-05-18  Kenya             72              45               61                59.3          74.2
    Why normalize per country, not globally?
    Nigeria has a much larger internet population than Ghana. Raw interest scores would make Nigeria look dominant in every single week — not because demand is higher, but because the search volume baseline is higher. Normalizing per country makes the score reflect trend direction within each country. You are asking "is demand rising in Ghana?" not "does Ghana have more searches than Nigeria?" That is the correct analytical question for market expansion decisions.
