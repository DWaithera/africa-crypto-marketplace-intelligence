Session 1
Session 1 Set up your professional workspace 2 hours
  • 1
    Open Anaconda Navigator
    Launch Anaconda Navigator from your Start Menu (Windows). You should see the home screen with Jupyter Notebook listed. If you do not have Anaconda installed, download it from anaconda.com/download — choose the Python 3.x version.
  • 2
    Open a terminal inside Anaconda
    In Anaconda Navigator, click Environments on the left sidebar. Select base (root). Click the play button ▶ and choose Open Terminal.

    A black command window opens. This is where all your setup commands will run. Do not close it.
  • 3
    Install all project libraries
    Type each command below one at a time. Press Enter after each. Wait for the installation to complete before typing the next command. You will see a lot of text scroll past — that is normal.
    bash — terminal 
    pip install pandas numpy matplotlib seaborn
pip install requests pytrends statsmodels
pip install scikit-learn openpyxl xlrd
pip install jupyter notebook ipykernel
    Why these libraries?
    pandas + numpy — data manipulation, the foundation of everything
    matplotlib + seaborn — charts and visualisations
    requests — for calling the CoinGecko API (Day 2)
    pytrends — for pulling Google Trends data (Day 3)
    statsmodels — for the correlation and trend analysis (Week 3)
    scikit-learn — for the scoring model (Week 4)
    You are installing everything now so you never hit a missing-library error mid-session.
  • 4
    Create your project folder structure
    In the same terminal window, navigate to your Documents folder and create the project. Type each line and press Enter. Do not skip any.
    bash — terminal 
    cd Documents
mkdir africa-crypto-intelligence
cd africa-crypto-intelligence
mkdir data notebooks visuals reports dashboard docs
  • 5
    Verify the folder structure exists
    Type this and press Enter:
    bash — terminal 
    ls
    Expected output
    data   notebooks   visuals   reports   dashboard   docs
    If you see all 6 folders listed, your workspace is ready. If any folder is missing, re-run the mkdir command for that specific folder.
    ⚠ If you get "command not found" on ls (Windows): Use dir instead. Windows Command Prompt uses dir while Mac/Linux uses ls. Both do the same thing.
  • 6
    Understand what each folder is for
    Before you move on, know your own structure:

    data/ — all raw and cleaned CSV files live here. Never put notebooks here.
    notebooks/ — all your Jupyter notebooks. This is where you write code.
    visuals/ — every chart or image you generate with Python (PNG files).
    reports/ — written analysis, findings, final report.
    dashboard/ — Power BI file (.pbix) and screenshot exports (Week 4).
    docs/ — reference documents, data dictionaries, methodology notes.
    Mentor note — why structure matters
    Hiring managers open your GitHub before they interview you. The first thing they see is your folder structure. A clean, logical structure tells them: this person thinks like an engineer. A single messy folder full of untitled notebooks tells them: this person has not worked in a professional environment. Today you are already ahead of 70% of applicants by doing this one thing correctly.

Session 2 Create your GitHub repository and push your first commit 2 hours
  • 1
    Go to github.com and create the repository
    Open your browser. Go to github.com. Sign in to your account (DWaithera).

    Click the green New button (top left of the dashboard). Fill in exactly as below:

    Repository name: africa-crypto-intelligence
    Description: Africa Crypto Market Intelligence & Expansion System — Analytics project identifying optimal exchange expansion markets across Kenya, Nigeria, Ghana, and South Africa.
    Visibility: Public
    Do NOT tick: "Add a README file" — we create it manually
    Do NOT tick: Add .gitignore or Choose a license — we handle these later

    Click Create repository.
  • 2
    Verify Git is installed on your machine
    Open your terminal (same one from Session 1, or open a new one from Anaconda). Type:
    bash — terminal 
    git --version
    Expected output
    git version 2.x.x
    If you get an error instead of a version number, go to git-scm.com, download and install Git for your operating system, then restart your terminal and try again.
  • 3
    Configure Git with your identity
    This only needs to be done once ever on your machine. Replace the email below with the email address you used to sign up to GitHub:
    bash — terminal 
    git config --global user.name "Damaris Waithera"
git config --global user.email "your-github-email@example.com"
    ℹ These commands tell Git who you are. Every commit you push will be signed with this name.
  • 4
    Navigate to your project folder in the terminal
    bash — terminal 
    cd Documents/africa-crypto-intelligence
    Confirm you are in the right place by typing pwd (Mac/Linux) or cd (Windows). You should see the path ending in africa-crypto-intelligence.
  • 5
    Create the README.md file
    First, create the file with this command:
    bash — terminal 
    echo "# Africa Crypto Market Intelligence System" > README.md
    Now open the README.md file in Notepad (Windows: right-click → Open with → Notepad) or any text editor. Delete everything inside and replace it with this content exactly — type it out, do not paste:
    markdown — README.md 
    # Africa Crypto Market Intelligence System

## Project Overview
A data analytics system that scores and ranks African markets
(Kenya, Nigeria, Ghana, South Africa) for cryptocurrency exchange
expansion using demand signals, payment infrastructure data,
and marketplace activity proxies.

## Business Question
Where and when should a crypto exchange expand next in Africa?

## Final Output
Expansion Opportunity Score (EOS) — a composite metric ranking
4 countries for exchange market entry.

## Data Sources
- CoinGecko API (BTC/ETH price and volume)
- Google Trends via pytrends (crypto search demand)
- Central Bank of Kenya (M-Pesa transaction data)
- World Bank financial inclusion data

## Tech Stack
Python · pandas · matplotlib · seaborn · statsmodels · Power BI

## Author
Damaris Waithera | Data Analyst | Nairobi, Kenya
GitHub: https://github.com/DWaithera
    Save the file. Close Notepad.
    Mentor note — the README is your cover letter
    When a recruiter at Binance, OKX, or Yellow Card clicks your GitHub link, the README is the first thing they read. It must answer: what is this? why does it matter? what does it produce? The README you are writing today does all three. Add your LinkedIn URL once you update your profile tomorrow. Do not leave that line blank forever.
  • 6
    Initialise Git and push everything to GitHub
    In your terminal (confirm you are inside africa-crypto-intelligence), type each command one at a time:
    bash — terminal 
    git init
git add .
git commit -m "Initial commit: project structure and README"
git branch -M main
git remote add origin https://github.com/DWaithera/africa-crypto-intelligence.git
git push -u origin main
    Git may ask for your GitHub username and password on the push command. Enter them. If you get a credentials error, GitHub may require a Personal Access Token (PAT) instead of your password — go to github.com → Settings → Developer Settings → Personal Access Tokens → Tokens (classic), generate a token with "repo" scope, and use that as your password.
    ⚠ Common error: "remote origin already exists" — If you see this, type: git remote remove origin first, then re-run the git remote add origin line and push again.
